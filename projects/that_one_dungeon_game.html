Overview

That One Dungeon is a roguelike dungeon crawler shipped commercially on Steam, built entirely in Godot at Blue Torch Games over three years. It was a live production application with real users, real reviews, and real release cycles.

The engineering challenges here translate directly to backend work: deterministic state management, performance optimization under tight constraints, code review standards across releases, and CI workflows that prevented regressions from shipping. These are the same disciplines required for production distributed systems.

The project shipped and maintained applications across desktop platforms, including three operating systems: Windows, MacOS, and Linux, handling active players with no dedicated ops team — which meant building systems that were reliable by design, not by manual intervention.

Engineering Highlights

🏗️
OOP Architecture
  • Entity component system design
  • Deterministic state for reproducible runs
  • Separation of game logic from rendering
  • Modular systems for extensibility
⚡
Performance
  • Profiling and optimization of hot paths
  • Reduced instability during high-load execution
  • Memory management for long sessions
  • Batch rendering for entity-heavy scenes
🔄
Release Engineering
  • CI workflows across multi-release cycles
  • Code review standards for regressions
  • Platform builds: Steam, Google Play
  • Hotfix deployment process
